# バトル操作一覧

このファイルに書かれた操作をもとに、Claude がコードを実装します。
変更したい場合は、該当箇所を直接編集してください。

## 読み方

```
【状態】どのフェーズ・状況のとき
【対象】何をタップ/操作するか
【操作】タップ、長押し、スワイプ、ドラッグ など
【結果】何が起きるか
【演出A】ターンプレイヤーの画面でどんな演出が出るか（なし＝演出不要）
【演出B】非ターンプレイヤーの画面でどんな演出がでるか（なし＝演出不要）
【演出AB共通】AB共に同じ演出（なし＝演出不要）

```

## 1. マリガン（ゲーム開始直後）

### 1-1. 手札カードをタップ
- 【状態】マリガン画面表示中
- 【対象】手札の5枚のカード
- 【操作】タップ
- 【結果】カード詳細画面が開く（拡大画像+効果テキスト）
- 【演出AB共通】カードが配られる演出

### 1-2. 「このままゲームを始める」ボタン
- 【状態】マリガン画面表示中
- 【操作】ボタンタップ
- 【結果】手札を確定、セキュリティ5枚セット、バトル開始
- 【演出A】「自分のターン開始」演出
- 【演出B】「相手のターン開始」演出

### 1-3. 「引き直す」ボタン
- 【状態】マリガン画面表示中（1回のみ）
- 【操作】ボタンタップ
- 【結果】手札をデッキに戻し、シャッフルして5枚引き直し
- 【演出AB共通】カードが配られる演出

## 2. アクティブフェイズ

### 2-1. レスト状態のカードあり
- 【状態】アクティブフェイズ開始
- 【対象】なし（自動処理）
- 【操作】なし
- 【結果】レスト状態のカードをすべてアクティブに戻す
- 【演出A】フェーズ名表示 → カードが縦に戻るアニメ → 自動で次フェーズへ
- 【演出B】相手のフェーズ名表示 → 相手のカードが縦に戻るアニメ → 自動で次フェーズへ
- 【例外】
    - 「アクティブにならない」効果を受けているデジモンはアクティブにしない

### 2-2. レスト状態のカードなし
- 【状態】アクティブフェイズ開始
- 【対象】なし（自動処理）
- 【操作】なし
- 【結果】何もせずスキップ
- 【演出A】「アクティブフェイズ スキップ！」→ 自動で次フェーズへ
- 【演出B】なし → 自動で次フェーズへ

## 4. 育成フェイズ

※ どれか1つの操作をしたら、自動でメインフェイズへ移行

### 4-1. デジタマを孵化する
- 【状態】育成フェイズ中、育成エリアが空、デジタマデッキに1枚以上
- 【対象】育成エリア（デジタマの裏面画像が表示されている）
- 【操作】タップ
- 【結果】デジタマデッキの上から1枚を育成エリアの隣に置く
- 【演出AB共通】孵化演出（フラッシュ＋カード画像＋「〇〇 孵化！」）→ メインフェイズへ

### 4-2. 育成エリアのデジモンをバトルエリアへ移動
- 【状態】育成フェイズ中、育成エリアにLv.3以上のデジモンがいる
- 【対象】育成エリアのカード
- 【操作】長押し（0.4秒）→ スワイプ
- 【結果】バトルエリアに移動
- 【演出A】カードがスライド移動 → メインフェイズへ
- 【演出B】バトルエリアへ移動！（下に小さくカード名）

### 4-3. 育成エリアのデジモンが移動不可（Lv.2デジタマ）
- 【状態】育成フェイズ中、育成エリアにLv.2のデジタマがいる
- 【対象】育成エリアのカード
- 【操作】長押し＋スワイプ
- 【結果】移動しない。「🚨デジタマは移動できません」メッセージ表示
- 【演出A】注意マーク表示
- 【演出B】なし

### 4-4. パス

- 【対象】「パス → メインフェイズへ」ボタン
- 【操作】タップ
- 【結果】何もせずメインフェイズへ
- 【演出AB共通】「育成フェイズ スキップ！」

## 5. メインフェイズ

### 5-1. 手札カードをタップ（選択）
- 【状態】メインフェイズ中
- 【対象】手札のカード
- 【操作】タップ（クリック）
- 【結果】カードを選択状態にする（浮き上がり）+ 詳細画面
- 【演出AB共通】カードが上に浮く、ボタン選択時にボタンが浮かぶ+枠がつく

### 5-2. 手札カードをドラッグ → 空きスロットにドロップ（登場）
- 【状態】メインフェイズ中
- 【対象】手札のデジモン/テイマーカード
- 【操作】ドラッグ → バトルエリアの空きスロットにドロップ
- 【結果】カードを登場。登場コスト分メモリー消費
- 【演出AB共通】DIGITAL APPEAR 演出（フラッシュ＋カード表示）
- 【備考】
　　- Lv.2デジタマは登場不可。
　　- 進化専用カードも登場不可。
　　- 【登場時】効果がある場合は、カード詳細画面+「効果発動しますか？」のメッセージ+「はいorいいえ」のボタン表示。
　　- タイプ「オプション」は登場ではなく「使用」カテゴリ

### 5-3. 手札のオプションカードをドラッグ → 空きスロットにドロップ（使用）
- 【状態】メインフェイズ中
- 【対象】手札のオプションカード
- 【操作】ドラッグ → バトルエリアの空きスロットにドロップ
- 【結果】カードを使用。仕様コスト分メモリー消費
- 【演出A】オプション効果発動の演出 → 効果内容のポップアップ（数秒） → 対象選択 → カード詳細画面+「このカードでいいですか？」のメッセージ+「はいorいいえ」のボタン表示
          - 「はい」→ 各効果発動演出
        　- 「いいえ」→ 対象選択に戻る
- 【演出B】オプション効果発動の演出 → 相手：カード名+効果内容+効果発動中のメッセージ（Aが「はいorいいえ」を押すまで表示）
　　　　　- 「はい」 → 各効果発動演出
         - 「いいえ」→ 対象選択に戻る
- 【備考】
　　- オプションカードではアタックできない。
　　- 効果使用したらトラッシュへ
- 【例外】
    - 効果によっては、バトルエリアに残る場合もある。

### 5-4. 手札カードをドラッグ → バトルエリアのカードにドロップ（進化）
- 【状態】メインフェイズ中
- 【対象】手札の進化カード → バトルエリアのデジモン
- 【操作】ドラッグ → 対象デジモンの上にドロップ、1ドロー（自動処理）
- 【結果】進化。進化コスト分メモリー消費。進化元はスタックに
- 【演出AB共通】ドローの際にドローフェイズと同じ演出（カードが手札にくる演出（引いたカードの画像が画面に出る＆それがレベル6以上なら派手な演出にしたいかも…）、DIGITAL EVOLUTION 演出（回転＋フラッシュ＋カード変化）、レベル6以上に進化する時は派手にしたい…
- 【備考】進化条件を満たしている必要あり。【進化時】効果がある場合は、「効果発動しますか？」のメッセージと「はいorいいえ」のボタン表示
- 【例外】
    - 「進化できない」効果を受けている場合や進化条件を満たしていない場合は、「🚨進化できません‼」のメッセージ

### 5-5. 手札カードをドラッグ → 育成エリアのカードにドロップ（育成エリア進化）
- 【状態】メインフェイズ中
- 【対象】手札の進化カード → 育成エリアのデジモン
- 【操作】ドラッグ → 育成エリアにドロップ、1ドロー（自動処理）
- 【結果】育成エリアのデジモンが進化。進化コスト分メモリー消費
- 【演出AB共通】DIGITAL EVOLUTION 演出（回転＋フラッシュ＋カード変化）
- 【備考】【進化時】効果は発動しない。＜育成＞の記載がある場合のみ、効果発動する（育成【進化時】の場合は、進化したタイミングで「効果発動しますか？」のメッセージと「はいorいいえ」のボタン表示。

### 5-6. 自分のバトルエリアのカードをタップ（選択）
- 【状態】メインフェイズ中
- 【対象】自分のバトルエリアのデジモン
- 【操作】タップ
- 【結果AB共通】選択状態にする（枠ハイライト）+ 詳細画面表示
- 【演出】枠が光る

### 5-7. 相手のバトルエリアのカードをタップ
- 【状態】メインフェイズ中
- 【対象】相手のバトルエリアのデジモン
- 【操作】タップ
- 【結果AB共通】詳細画面表示
- 【演出】なし

### 5-8. アタック
- 【状態】メインフェイズ中（もしくは条件による「ターン終了時など記載があればその限りではない）
- 【対象】アクティブ状態のデジモン→レスト状態の相手のデジモンorセキュリティ
- 【操作】PC画面：長押し→「アタック」ボタン／スマホ画面：左にスワイプ→「アタック」ボタン
- 【結果】デジモンがレスト。アタック宣言→セキュリティチェック or バトル
- 【演出A】ドラッグ（スワイプ）すると、対象まで矢印が出現 → 対象確定 → アタック演出（VS画面表示）
- 【演出B】アタック演出（VS画面表示）
- 【備考】
    - レスト状態のカードではアタック不可。（「アタック」ボタングレーアウト＆押せなくなる）
    - このターンで登場したデジモンはアタックできない。    
    - 【アタック時】効果があれば、全ての効果処理が終わって（誘発も含め）からアタック演出
    - アクティブ状態のデジモンにはアタックできない。
    - タイプが「デジモン」以外のカードではアタックできない。
- 【例外】
    - 効果で「レスト状態でアタックできる」と記載あれば、レスト状態のデジモンでもアタック演出可能。
    - 効果で「レストせずアタックできる」と記載あれば、レストせずアタック演出可能。
    - 効果で【速攻】がついていれば、登場したターンでもアタック可能
    - 効果によってはアクティブ状態のデジモンにもアタック可能（【突進】がついているorアクティブ状態のデジモンにもアタック可能という記載があるetc…）
    - 効果によって、テイマーでもアタックできる場合がある。 

### 5-9. アタック終了
- 【状態】メインフェイズ中、アタック演出終了
- 【対象】今アタックしていたデジモン
- 【操作】なし
- 【結果】なし
- 【演出AB共通】生き残り→「Win‼（緑）」の演出＆バトルエリアにレスト状態で残る／引き分け（DPが同じ）or負け→「Lost（赤）」と消滅する演出＆バトルエリアからカードが消える
- 【演出A画面表示内容】
　　生き残り→「Win‼（緑）」／引き分け（DPが同じ）or負け→「Lost（赤）」／テイマー→「テイマー登場！」+カード画像+効果内容表示／オプション→セキュリティ効果発動！+カード画像+効果内容表示
- 【演出B画面表示内容】
　　セキュリティデジモンの方がDP上→「Win‼（緑）」／引き分け（DPが同じ）orセキュリティデジモンの方がDP低い→「Lost（赤）」テイマー→「テイマー登場！」+カード画像+効果内容表示／
　　オプション→セキュリティ効果発動！+カード画像+効果内容表示
- 【備考】【アタック終了時】効果があれば、効果処理発動

### 5-10. ブロック
- 【状態】メインフェイズ中、相手のアタック宣言時
- 【対象】【ブロッカー】効果を持っているデジモン
- 【操作】タップ
- 【結果】デジモンがレスト→アタック対象がこのデジモンに強制的に移る→バトル
- 【演出A】アタック→ブロック確認のメッセージ（相手がボタン選択するまで表示）
- 【演出B】相手がアタック→ブロックしますか？のポップアップ（はいorいいえ）
　　　　　 ブロッカーが2体以上いる場合：対象選択→これでいいですか？（はいorいいえ）／「はい」の場合：選択したデジモンでブロック、「いいえ」の場合：対象選択に戻る
- 【備考】 生き残り→「Win‼（緑）」の演出、負け→「Lost（赤）」と消滅する演出

### 5-11. セキュリティチェック
- 【状態】メインフェイズ中、相手のデジモンが自分のセキュリティにアタック成立
- 【対象】セキュリティーの一番上
- 【操作】なし（自動でセキュリティの一番上のカードが表向きになる）
- 【結果】表向きになる（カード画像表示）、デジモンならバトル発生、オプションならセキュリティ効果発動、テイマーならバトルエリアに登場させる。
- 【演出AB共通】裏向きのカードが表向きになる演出、デジモンならバトル演出（負けたら消滅演出）、オプションなら魔法使う演出、テイマーなら登場演出
- 【備考】 両画面でセキュリティ枚数が減る

### 5-12. ターン終了
- 【状態】メインフェイズ中
- 【対象】「⏭ ターン終了」ボタン
- 【操作】タップ
- 【結果】ターン終了。相手のメモリー3でターン移行
- 【演出A】「自分のターン終了」演出
- 【演出B】相手がターンを終了しました。のメッセージが流れる→「相手のターン終了」演出
- 【例外】
    - 【自分のターン終了時】効果を持っているカードがあれば、全ての効果処理が終わってから「自分のターン終了」演出

### 5-13. 効果発動（自分のバトルエリアのカード）
- 【状態】メインフェイズ中、効果発動条件を満たした時
- 【対象】長押し（左スワイプ）→「効果」ボタン押下or条件満たした時に即時で自動発動
- 【操作】ボタンタップ
- 【結果】強制の場合：効果発動／任意の場合：カード詳細画面+「効果を発動しますか？」表示→効果発動
- 【演出A】強制：対象選択or各効果発動演出／任意：カード詳細画面+「効果を発動しますか？」表示。「はい」で発動（「いいえ」だと効果発動せず元の状態に戻る）
- 【演出B】カード名+効果の内容表示（相手が「はいorいいえ」を押すまで表示する）
- 【備考】発動したら「効果」ボタンはグレーアウト＆押せなくなる（複数発動可能なものは除く）

### 5-14. トラッシュのカード確認
- 【状態】メインフェイズ中
- 【対象】トラッシュエリア
- 【操作】相手or自分のトラッシュエリアタップ
- 【結果】トラッシュにあるカードの詳細確認
- 【演出AB共通】トラッシュにあるカード一覧表示→カードタップで詳細画面表示

### 5-15. 相手のターン開始
- 【状態】相手のターン開始時演出が終わった
- 【対象】【相手のターン開始時】効果を持っているカード
- 【操作】なし（自動処理）
- 【結果】強制：効果発動／任意：「効果を発動しますか？」確認ダイアログ表示→「はい」で発動（「いいえ」だとアクティブフェイズに入る）
- 【演出A】5-13と同じ
- 【演出A】5-13と同じ

### 5-16. 自分のターン開始
- 【状態】自分のターン開始時演出が終わった
- 【対象】【自分のターン開始時】効果を持っているカード
- 【操作】なし（自動処理）
- 【結果】強制：効果発動／任意：「効果を発動しますか？」確認ダイアログ表示→「はい」で発動（「いいえ」だとアクティブフェイズに入る）
- 【演出A】5-13と同じ
- 【演出A】5-13と同じ

### 5-17. ゲートを退出
- 【状態】バトル画面中いつでもOK
- 【対象】ルーム全体
- 【操作】「ゲートを退出する」ボタンを押下
- 【結果】バトル画面を途中で退出する（デッキ選択画面に戻る）
- 【演出A】退室確認　ゲートを出ますか？バトルの進行状況は失われます。（はいorいいえ）が表示される
　　　　　「はい」押下：デッキ選択画面に戻る／「いいえ」押下：バトル画面に戻る
- 【演出B】相手が「はい」を押した場合：「途中退室　「ユーザー名」が途中退室しました。バトルは続行できません。」+ゲートを退出ボタン表示

- 【結果】詳細画面を閉じる

## 7. メモリー超過（自動）

### 7-1. メモリー表示の基本
- 先攻=赤／後攻=青（「自分」「相手」記載の文字も同様ルール）
- 【AB画面共通】自分のメモリーが左側、相手のメモリーが右側表示
- 【A画面】メモリー+N：右側へ移動／メモリー-N：左側へ移動
- 【B画面】メモリー+N：左側側へ移動／メモリー-N：右側へ移動

### 7-2. メモリーが1以下になった
- 【状態】登場/進化コスト支払い後
- 【結果】メモリーの絶対値が相手のメモリーになり、ターン移行
- 【演出】「自分のターン終了」→「相手のターン開始」演出

## 変更したいときは？

上の各項目を直接書き換えてください。例えば：

- 操作を変えたい → 【操作】を書き換え
- 演出を変えたい → 【演出】を書き換え
- 新しい操作を追加 → 同じフォーマットで追記
- 操作を削除 → 該当項目を削除 or 「※無効」と書く

例：
```
### 5-99. 新しい操作名
- 【状態】どんなとき
- 【対象】何を
- 【操作】どうする
- 【結果】何が起きる
- 【演出】どんな演出
```

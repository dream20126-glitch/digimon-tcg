# 効果処理システム設計書（辞書方式）

## 概要
- カードの効果テキストはそのまま使う（効果スクリプト列は不要）
- スプレッドシートの「効果辞書」シートにキーワードと処理ルールを管理
- コードが効果テキストと辞書を照合して自動処理
- 新効果の追加はスプレッドシートに行追加するだけ（コード変更不要）

## 処理の流れ

```
1. カードの効果テキストを取得
   例:「【登場時】1ドローすることで、相手のデジモン1体のDPを-3000する」

2. 効果辞書からトリガーを特定
   「登場時」→ 辞書ヒット → 即時トリガー

3. 強制/任意を判定
   「ことで」→ 辞書ヒット → 任意発動

4. コスト部分を特定
   「1ドロー」+「ことで」→ コスト: draw,1

5. アクション部分を特定
   「DPを-3000」→ アクション: dp_minus,3000

6. 対象を特定
   「相手のデジモン1体」→ 対象: opponent_select,1

7. 実行
   任意 → 「効果発動しますか？」→ はい →
   コスト実行（1ドロー）→ 対象選択 → アクション実行（DP-3000）
```

## コードが参照する列

効果辞書シートの各列の意味と、コードがどう使うか。

| 列名 | コードでの使い方 |
|---|---|
| キーワード | 効果テキスト内を検索するパターン。{N}は数値のワイルドカード |
| 分類 | トリガー/アクション/キーワード/判定/コスト のどれか |
| 処理名 | コードが実行する処理の名前（draw, dp_plus, blocker等） |
| 発動分類 | 即時/永続/パッシブ/割り込み のどれか |
| デフォルト対象 | 対象指定がなければこの対象を使う |
| デフォルト持続 | 持続指定がなければこの持続を使う |
| 備考 | コードは参照しない。人間用メモ |

## コード側の処理ルール

### トリガー判定
1. 効果テキストから【】で囲まれた部分を取得
2. 辞書で「分類=トリガー」の行を検索
3. マッチしたら、その行の「発動分類」で処理分岐

### 強制/任意判定
1. 効果テキスト内に辞書の「分類=判定」キーワードがあるかチェック
2. 「ことで」「できる」「してもよい」等があれば任意
3. なければ強制

### コスト判定
1. 「ことで」「することで」の前の部分からコストを抽出
2. 辞書の「分類=コスト」でマッチング

### アクション判定
1. コスト部分以降（or 全体）から辞書の「分類=アクション」を検索
2. {N}部分は正規表現で数値を抽出
3. 複数アクションがある場合はテキスト順に実行

### 対象判定
1. 効果テキストから「相手のデジモンN体」等を検索
2. 辞書の対象パターンでマッチング
3. マッチしなければデフォルト対象を使用

## 新しい効果への対応

### スプレッドシートに行追加だけで対応
- 既存のアクション処理の組み合わせ
- 新しいキーワード（辞書に行追加）
- 数値の変更

### コード追加が必要
- 全く新しいアクションタイプ（例: deck_open, add_to_evo_source）
  → コードに処理関数を1つ追加し、辞書から呼び出せるようにする

# 効果処理システム設計書

## 全体の流れ

```
※A画面＝ターンプレイヤー／B画面＝非ターンプレイヤー

効果発動トリガー発生（登場、進化、アタック、メイン操作 等）
  ↓
① カード情報一覧の「レシピ」列を参照
  ↓
② レシピ内のトリガーキーで処理を特定（効果辞書参照）
   （on_play, on_evolve, on_attack, main, security, while_in_security 等）
  ↓
③ 任意効果なら「効果を発動しますか？」確認
   → はい → ④へ
   → いいえ → 相手に「発動しませんでした」通知 → 終了
  ↓
④ 両方の画面に効果内容を表示
   A画面: カード名＋効果テキスト（効果テキスト列から）
   B画面: 同じ内容のオーバーレイ（効果処理完了まで表示）
  ↓
⑤ レシピのステップを順番に実行
   各ステップで効果アクション辞書を参照
   → アクションの演出タイプ・Bの画面演出を辞書から取得
  ↓
⑥ 対象選択が必要なアクションの場合
   A画面: 対象選択UI（タップで選択 → 確認ダイアログ）
   B画面: 効果内容オーバーレイ表示中（Aが「はい」を押すまで）
  ↓
⑦ Aが対象を確定（「はい」を押す）
   → B画面の効果内容オーバーレイが消える
  ↓
⑧ アクション実行＋演出再生
   A画面: 演出再生（演出タイプに基づく）
   B画面: 同じ演出を再生（Bの画面演出列に基づく）
   カード除去時: card_removed / own_card_removed コマンドで即時同期
  ↓
⑨ state_sync送信（盤面全体の整合性確保）
  ↓
⑩ メモリー変動があれば memory_update で即時同期
```

## データの参照関係

```
カード情報一覧
  ├─ キーワード列     → コードが直接参照（ブロッカー、突進等の判定）
  ├─ 効果テキスト列   → 画面表示用（Bの効果内容オーバーレイにも使用）
  ├─ 進化元テキスト列 → 画面表示用
  ├─ セキュリティテキスト列 → 画面表示用
  └─ レシピ列         → 効果エンジンが実行するJSON
        │
        ├─ トリガーキー（on_play, main, security 等）
        │    └─ 効果辞書で定義（キーワード → 処理コード）
        │
        └─ アクションステップ（destroy, dp_plus 等）
             └─ 効果アクション辞書で定義
                  ├─ アクション名（テキストマッチ用）
                  ├─ アクションコード（実行処理の識別）
                  ├─ 演出タイプ（A画面の演出）
                  └─ Bの画面演出（相手画面の演出）
```

## レシピ自動生成の流れ（GAS）

```
カード情報一覧の効果テキスト列を読む
  ↓
① キーワード抽出
   効果テキストから【ブロッカー】【突進】等を検出 → キーワード列に書き込み
  ↓
② トリガー特定（効果辞書を参照）
   効果テキスト内の【】で囲まれたキーワードを効果辞書で検索
   → 処理コードを取得（on_play, on_evolve, main 等）
  ↓
③ アクション特定（効果アクション辞書を参照）
   効果テキスト内のアクションキーワードを検索
   → アクションコードを取得（destroy, dp_plus 等）
   → 数値（{N}）も抽出
  ↓
④ 対象・条件・持続を特定（効果アクション辞書を参照）
   target_opponent, cond_no_evo_source, dur_this_turn 等を検索
  ↓
⑤ 任意/強制を判定（効果アクション辞書を参照）
   judge_optional に該当するキーワードがあるか
  ↓
⑥ セキュリティテキスト列も同様に解析
   → セキュリティテキスト列の【セキュリティ】→ "security"（めくれた時）
   → 効果テキスト列の【セキュリティ】→ "while_in_security"（ある間）
  ↓
⑦ 進化元テキスト列も同様に解析
   → "evo_source" キー内にトリガーとアクションを格納
  ↓
⑧ JSON生成 → レシピ列に書き込み
```

## 効果辞書の列構成

| 列名 | 用途 | 例 |
|---|---|---|
| キーワード | テキストマッチ用 | 【登場時】、【メイン】 |
| 処理コード | トリガーの識別コード | on_play, main, security |
| 持続時間 | デフォルトの持続 | dur_this_turn |
| 処理名 | 特殊処理の識別 | per_count |
| 進化元対応 | 進化元でも発動するか | はい / いいえ |
| 備考 | 人間用メモ | — |

### セキュリティのトリガー（2種類）

| キーワード | 処理コード | 説明 |
|---|---|---|
| 【セキュリティ】 | security | セキュリティテキスト列：めくれた時に1回発動 |
| 【セキュリティ】 | while_in_security | 効果テキスト列：セキュリティにある間ずっと有効 |

※同じキーワードだが、GASが「どの列から見つけたか」で処理コードを振り分ける

## 効果アクション辞書の列構成

| 列名 | 用途 | 例 |
|---|---|---|
| アクション名 | テキストマッチ用（カンマ区切り可） | 消滅させる、消滅する |
| アクションコード | アクションの識別コード | destroy |
| 演出タイプ | A画面で使う演出 | 消滅演出 |
| 枠色 | 対象選択時の枠色 | 赤 |
| Bの画面演出 | 相手画面の演出 | Aと同じ |

## カード情報一覧の列構成

| 列名 | 用途 |
|---|---|
| カードNo | カード識別子 |
| 名前 | カード名 |
| タイプ | デジモン/テイマー/オプション/デジタマ |
| 色 | 赤/青/黒 等 |
| レベル | 2〜6 |
| 登場コスト | 数値 |
| 進化条件 | 赤Lv.3 等 |
| 進化コスト | 数値 |
| DP | 数値 |
| 特徴 | 竜型 等 |
| キーワード | ブロッカー,突進 等（カンマ区切り、GAS自動生成） |
| 効果テキスト | 画面表示用の効果全文 |
| 進化元テキスト | 画面表示用の進化元効果全文 |
| セキュリティテキスト | 画面表示用のセキュリティ効果全文 |
| レシピ | JSON形式の効果定義（GAS自動生成） |
| ImageURL | カード画像リンク |

## レシピJSONの構造

```json
{
  "on_evolve": [
    {"action": "select", "target": "opponent", "store": "A"},
    {"action": "destroy", "card": "A"}
  ],
  "evo_source": {
    "keywords": ["Sアタック+1"],
    "on_attack": [
      {"action": "trash_evo_bottom", "card": "A", "count": 1}
    ],
    "permanent": {
      "condition": "own_turn",
      "dp_plus": 1000
    }
  },
  "security": [
    {"action": "use_main_effect"}
  ],
  "while_in_security": [
    {"action": "security_dp_buff", "value": 7000}
  ]
}
```

## オンライン対戦時の同期

### 共通ルール（全効果統一、コードで直接指定）
| タイミング | Aの画面 | Bの画面 |
|---|---|---|
| 効果発動 | 効果テキスト表示（2.5秒） | 効果内容オーバーレイ（処理完了まで表示） |
| 対象選択中 | 選択UI | 効果内容オーバーレイ表示中 |
| 対象確定（はい） | — | 効果内容オーバーレイ消去 |
| 任意効果で「いいえ」 | — | 「発動しませんでした」通知（2秒） |
| メモリー変動 | ゲージ即時移動 | ゲージ即時移動（memory_update） |
| カード除去 | 即時反映 | 即時反映（card_removed / own_card_removed） |

### アクションごとの演出（効果アクション辞書で管理）
| アクション | Aの画面演出 | Bの画面演出（辞書の「Bの画面演出」列） |
|---|---|---|
| destroy | 消滅演出 | Aと同じ |
| dp_plus/minus | 数値ポップアップ | Aと同じ |
| summon | 登場演出 | Aと同じ |
| memory_plus/minus | ゲージ移動 | ゲージ移動（自動） |
| bounce | — | カード名+結果表示 |
| rest | — | カード名+結果表示 |
| deck_open | カードめくり演出 | デッキオープン表示 |

### コードで直接指定する共通演出（辞書に入れない）
| 演出 | タイミング |
|---|---|
| 登場演出 | カードを場に出した時（全カード共通） |
| 進化演出 | 進化した時（全カード共通） |
| セキュリティチェック | セキュリティアタック時 |
| ダイレクトアタック | セキュリティ0でアタック時 |
| ブロック確認 | ブロッカーがいる時 |
| 勝利/敗北 | ゲーム終了時 |

## 新しいカード/効果への対応

| 追加するもの | 作業場所 | コード修正 |
|---|---|---|
| 新しい効果の文面パターン | 効果辞書 or 効果アクション辞書に行追加 | 不要 |
| 新しいキーワード能力 | 効果辞書に行追加 + GASのKEYWORD_PATTERNS追加 | 不要 |
| 新しいアクションの演出設定 | 効果アクション辞書のBの画面演出列を編集 | 不要 |
| 全く新しいアクション種類 | 効果アクション辞書に行追加 + コードに処理関数追加 | 必要 |
| 全く新しいゲームメカニクス | コードに処理追加 | 必要 |
